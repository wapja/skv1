## Task 21: GitHub Template repo + verify

**Steps:**

- [ ] **Step 21.1: Push to GitHub**

```bash
gh repo create wapja/skv1 --public --source . --remote origin
git push -u origin main
git push --tags
```

- [ ] **Step 21.2: Mark as Template**

Via GitHub UI: Settings → "Template repository" toggle ON. Or via API:
```bash
gh api repos/wapja/skv1 -X PATCH -F is_template=true
```

- [ ] **Step 21.3: Add GitHub secrets for CI**

```bash
gh secret set FLUX_PRO_USERNAME --repo wapja/skv1
gh secret set FLUX_PRO_PASSWORD --repo wapja/skv1
```

- [ ] **Step 21.4: Verify CI green**

Watch the first push trigger all 3 jobs (pint, pest unit/feature/arch, pest browser). Fix any CI-only failures (database hostnames, browser dependencies).

- [ ] **Step 21.5: Verify on a second clean machine**

```bash
# On a second laptop with Herd Pro installed
composer create-project wapja/skv1 verify-skv1
cd verify-skv1
herd link verify-skv1
herd secure verify-skv1
createdb -h 127.0.0.1 -U postgres verify_skv1
cp .env.example .env
sed -i '' 's/DB_DATABASE=skv1/DB_DATABASE=verify_skv1/' .env
php artisan key:generate
php artisan migrate --seed
npm install && npm run build
```

Open `https://verify-skv1.test/` — expect tenant-picker / login. Then run all 8 acceptance steps from super prompt §11. **Eén faal → fix in skv1, push, retry on verify machine.**

- [ ] **Step 21.6: Final commit + tag**

```bash
git tag v0.1.0
git push --tags
```
